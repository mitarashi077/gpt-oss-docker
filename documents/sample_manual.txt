GPT-OSS Docker RAG システム操作マニュアル

====================================================================

1. 環境の起動

   コマンド: scripts\start.bat
   
   このコマンドで以下が実行されます：
   - Docker コンテナのビルドと起動
   - Ollama モデルの自動ダウンロード
   - 各サービスの健全性チェック
   
   初回起動時は10-15分程度かかることがあります。

====================================================================

2. 文書のアップロード

   方法1: バッチファイルを使用
   コマンド: scripts\upload_doc.bat
   手順: プロンプトに従ってファイルパスを入力
   
   方法2: cURLコマンド
   curl -X POST "http://localhost:8001/upload" -F "file=@path/to/document.pdf"

====================================================================

3. 文書の検索

   方法1: バッチファイルを使用
   コマンド: scripts\search_doc.bat
   手順: プロンプトに従って検索クエリを入力
   
   方法2: cURLコマンド
   curl -X POST "http://localhost:8001/search?query=検索したいキーワード"

====================================================================

4. RAGチャットの利用

   コマンド: scripts\rag_test.bat
   
   対話モードが開始され、以下のコマンドが使用できます：
   - 通常の質問: そのまま入力
   - /health: サービス状態確認
   - /upload: 文書アップロード
   - /clear: データベースクリア
   - /quit: 終了

====================================================================

5. トラブルシューティング

   問題1: サービスが起動しない
   解決: Docker Desktop が起動していることを確認
   
   問題2: モデルダウンロードに時間がかかる
   解決: 初回は大きなファイルをダウンロードするため時間がかかります
   
   問題3: 検索結果が出ない
   解決: まず文書をアップロードしてください
   
   問題4: 日本語が文字化けする
   解決: UTF-8エンコーディングでファイルを保存してください

====================================================================

6. 環境の停止

   コマンド: scripts\stop.bat
   
   全てのDockerコンテナが安全に停止されます。
   データは保持されるため、再起動後も利用可能です。

====================================================================

7. サポートされるファイル形式

   - PDF (.pdf)
   - テキストファイル (.txt)
   - Markdown (.md)
   - その他のテキスト系ファイル

====================================================================

注意事項:
- 初回起動時はインターネット接続が必要です
- 大きなファイル（100MB以上）のアップロードは時間がかかります
- システム要件: 8GB以上のRAM推奨
